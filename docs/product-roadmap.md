# Product Roadmap

## Stage 0: Portfolio-safe Documentation

Goal: Present the project as a credible public case study for graduate applications and early collaborator review.

Deliverables:

- Public-safe README.
- Project overview.
- Architecture documentation.
- Workflow documentation.
- Synthetic sample output.
- Data governance principles.
- Evaluation rubric.
- SUTD DAI-E fit statement.

Status: Completed in this public repository.

## Stage 1: Intake Prototype

Goal: Build a structured intake flow for student and family planning information.

Planned work:

- Define intake fields.
- Separate hard constraints and soft preferences.
- Add missing-information prompts.
- Design privacy-aware data handling.
- Create advisor-readable profile output.

The immediate next milestone is to create a clickable intake prototype and one advisor-reviewed synthetic report.

## Stage 2: Sample Report Generator

Goal: Generate draft planning summaries from structured input using synthetic or consent-approved data.

Planned work:

- Produce conservative, balanced, and ambitious categories.
- Generate explanation notes.
- Add uncertainty labels.
- Include source provenance placeholders.
- Prepare outputs for advisor review.

## Stage 3: Advisor Review Workflow

Goal: Make human review part of the product flow.

Planned work:

- Create advisor checklist.
- Add edit and approval states.
- Track unresolved questions.
- Support revised summaries.
- Document review standards.

## Stage 4: Pilot Consultation Workflow

Goal: Test the workflow in a controlled advisory setting.

Planned work:

- Use consent-based or synthetic pilot profiles.
- Measure whether outputs improve consultation clarity.
- Collect advisor feedback.
- Refine intake and report structure.
- Improve source tracking and uncertainty disclosure.

## Stage 5: Enterprise / Consulting Deployment

Goal: Explore a deployable workflow for education consulting or advisory operations.

Planned work:

- Define access control and data retention policy.
- Separate private operational data from public materials.
- Improve audit logs and provenance tracking.
- Build dashboard support for advisors.
- Evaluate compliance and responsible AI requirements.

## Roadmap Principle

The project should grow from a documented workflow into a controlled advisory system. It should not skip privacy, human review, or evaluation in pursuit of faster automation.
